Core Simulation & Movement Interactions

Shared Physics Helpers
powder(x,y,i, slip, speedUp)
• Used by: SPARKLE, SHADOW, CRYSTAL, SALT, SEED, ASH, ICE, (VOID lazy)
• Down + optional diagonal slip. Density swap vs lower-density liquids.
• nearMud() check: 55% chance to refuse move when next to MUD.
• Falls off edges → deleted (void).

liquid(x,y,i, disperse, moveChance)
• Used by: WATER, MUD, HONEY, LAVA, SALTWATER
• Down first, then sideways spread + horizontal flooding.
• moveChance varies (HONEY uses neighbor count for stickiness).
• Edges = deletion.
Special Movement (not using powder/liquid)
• STONE: only straight down or swap with liquids below (no slip).
• DREAM: biased drift + phase-swap through almost everything.
• LIGHTNING: 4-step downward "zip" with random side jolts + glow.
• BOUNCE: internal velocity (life=vy, shade=vx), multi-cell sweeps, restitution bounce.
• BIRD: 8-dir state machine in shade, perching (life timer) on WOOD/BRICK/PLANT/VINE.
• FISH: must stay adjacent to WATER or SALTWATER or sinks/flops/suffocates.
• CLOUD: prefers rising, produces WATER below.
• VINE: support-aware hanging + single-tendril growth (avoids clumping).
Key Emergent Loops & Hubs
FIRE + LAVA are the biggest hubs (ignite plants/wood/honey, bake mud, melt ice/stone, fuse sparkle).

Water cycle: Cloud rains → pools → salt dissolves → brine evaporates near heat → salt crystals.
Life cycle: Seed + water/mud → Plant/Vine → can burn or frost back to seed/ash.
Lightning turns water into more lightning (chain) then plants on fire.
Void eats matter then "gives birth" to dream motes when it expires.
Lava ↔ Stone: Lava can melt stone into more lava; water can quench lava into stone.

UI → State → Simulation Flow

Input Layer
• Canvas pointer events (down/move/up/leave + capture)
• Brush slider + [ ] keys
• Rotary time dial (drag + wheel) → exponential 0.1×–3.0× mapped to simSpeed
• Palette clicks + number keys (and special letters) → current material
• Space = pause, C = clear
Painting
• Only writes into EMPTY cells (eraser always clears)
• Disk brush with per-material fill probability (liquids dense, dream/void/bird/fish sparse, fire medium)
• Special initial life[] / shade[] for FIRE, LIGHTNING, LAVA, BOUNCE
• paintStroke() does grid-space Bresenham interpolation
Tick & Render
• rAF tick() accumulates simSpeed into stepAcc
• 0–6 simulation steps per visual frame (capped)
• step() marks + updates bottom-up with row parity
• Global fade on glow[] and trailA[] every frame
• Render: offscreen 1:1 → upscale + additive puffs + overlay frame + cursor

Data Structures (how elements "talk")

All communication is direct array mutation inside the single update() pass.
cells[] (Uint8) — type
shade[] (Uint8) — color variation + overloaded (BIRD/FISH direction, BOUNCE vx, LIGHTNING "was water")
life[] (Int16) — countdown / growth energy / velocity / perch timer
glow[] (Float32) — light, written by many, read in render + some reactions (shadow swallows it)
trailA/H[] — DREAM only (color trails that linger after the mote moves)
parts[] — independent cosmetic puff particles (position/vel/life/hue). Emitted from 20+ reaction sites.

Boundaries & World Rules

The grid has no walls — anything that moves off any edge is deleted (falls into the void).
The seeded brick enclosure (BRICK walls + floor) is the only thing that normally contains the simulation.
Clear (C) removes walls too. Painting or destroying walls lets matter drain.
